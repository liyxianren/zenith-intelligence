# 让全世界看到你的作品

## 学习目标

- 理解网站部署的基本概念
- 认识 GitHub Pages 的用途
- 会把静态网页发布到网上

## 什么叫部署

部署就是把本地写好的网页放到服务器上，让别人通过网址访问。

## GitHub Pages 适合什么

- 个人主页
- 静态作品展示
- HTML/CSS/JS 小项目

## 基本步骤

1. 注册 GitHub 账号
2. 创建新仓库
3. 上传项目文件
4. 在仓库设置中启用 Pages
5. 获得公开网址

## 发布前检查

- 首页文件是否叫 `index.html`
- 资源路径是否正确
- 页面在手机和电脑上是否都能打开

## 小结

作品只有“被别人看到”，才真正进入反馈循环。部署不是结束，而是下一轮改进的开始。
